# Technical Documentation

## Project Overview

- Project name: `OneDayOld` (Unity project with Android build target)
- Unity version: `6000.3.13f1`
- Primary build target: `Android (API 13)`
- Main game type: Side-scrolling endless runner with gravity flip mechanics and obstacle avoidance
- Key gameplay mechanic: player flips gravity up/down to navigate hazards while the level continuously scrolls to the right

## Project Structure

- `Assets/`
  - `Scripts/` — core game logic and systems
  - `Maps/` — likely map-specific assets, such as scene or prefab collections
  - `Prefabs/` — reusable runtime objects
  - `Scenes/` — Unity scenes for menu/gameplay
  - `Settings/` — runtime settings assets
  - `TextMesh Pro/` — text rendering assets
  - `TutorialInfo/` — tutorial or editor helper assets
- `Packages/manifest.json` — Unity package dependencies and modules
- `Assembly-CSharp.csproj` — C# project configuration generated by Unity
- `ProjectSettings/` — Unity project configuration files
- `EndlessAdventure.sln` / `EndlessAdventure.slnx` — solution files for Visual Studio/IDE integration

## Core Systems

### Game State Management

#### `GameManager.cs`
- Singleton controlling overall game state
- Methods:
  - `StartGame()` — resumes gameplay by setting `Time.timeScale = 1`
  - `PauseGame()` — freezes gameplay with `Time.timeScale = 0`
  - `GameOver()` — pauses the game, saves progress, and signals UI
  - `RestartGame()` — resets the player and map, then resumes gameplay
  - `BackToMenu()` — resets the scene to a menu state and clears loaded segments

### Player Control

#### `PlayerMovement.cs`
- Uses Unity Input System for keyboard and pointer-based input
- Player moves horizontally automatically and flips vertical orientation on input
- Handles custom physics by disabling built-in gravity and applying manual vertical force
- Maintains bounds with `maxFlight` and `minFlight`
- Supports player reset via `ResetPlayer()`

### Level and Obstacle Management

#### `LevelManager.cs`
- Singleton responsible for dynamic segment spawning and obstacle recycling
- Uses a queue of active segments to recycle old map sections
- Automatically adjusts difficulty by increasing:
  - `currentSpeed`
  - `currentLaunch`
- Obstacle spawning settings include:
  - `obstaclesPerSegment`
  - `mapSegment` prefab
  - random placement along a segment
- `ClearMap()` resets state and spawns initial segments

#### `Obstacle.cs`
- Provides spawn type behavior via `Obstacle.SpawnType`
- Detects player proximity to trigger sounds or visual feedback
- Triggers game over when colliding with the player

### Map and Content Loading

#### `MapManager.cs`
- Loads level-specific data from `MapData` scriptable objects
- Handles map configuration:
  - `LoadObstacles()` → sets obstacle prefabs in `LevelManager`
  - `LoadSkyBox()` → updates RenderSettings skybox
  - `LoadPlayerCharacter()` — instantiates and parents the player character prefab

#### `MapData.cs`
- Scriptable object data model for map definitions
- Fields:
  - `mapName`
  - `skyboxMaterial`
  - `playerCharacter`
  - `backgroundGameObject`
  - `obstaclePrefabs[]`

### UI and Menu Flow

#### `UIManager.cs`
- Singleton managing all main menus, HUD, game over, settings, and map selection UI
- Supports:
  - menu navigation and map selection
  - play, pause, resume, and exit actions
  - showing game instructions briefly
  - transitioning between main menu and gameplay
  - awarding high score badges on game over
- Integrates with `AudioManager` for button clicks and gameplay sounds

### Audio System

#### `AudioManager.cs`
- Singleton managing music and sound effects
- Contains AudioSource references for music and SFX
- Supports:
  - `PlayMusic()` with looping background tracks
  - `PlaySFX()` for one-shot sound effects
  - pause/resume and stop controls

### Score Tracking and Unlocks

#### `DistanceTracker.cs`
- Singleton that tracks run distance and high score
- Updates live UI during gameplay
- Detects when high score is beaten and updates multiple UI text elements
- Manages unlock logic for map locks based on high score thresholds:
  - `map2Distance`, `map3Distance`, `map4Distance`, `map5Distance`
- Implements `IDataPersistence` to load/save high score data

### Data Persistence

#### `DataPersistenceManager.cs`
- Singleton responsible for loading and saving game data
- Uses `FileDataHandler` to serialize `GameData` to JSON
- Finds all active `IDataPersistence` objects in the scene and delegates load/save
- Saves automatically on `OnApplicationQuit()`

#### `FileDataHandler.cs`
- Handles low-level file I/O and optional XOR-based encryption
- Reads/writes JSON data from `Application.persistentDataPath`
- Error handling logs exceptions during load/save operations

#### `IDataPersistence.cs`
- Interface with two required methods:
  - `LoadData(GameData data)`
  - `SaveData(ref GameData data)`

#### `GameData.cs`
- Serializable model containing persistent game state
- Current saved state:
  - `highestDistanceCount`

### Settings

#### `VolumeSettings.cs`
- Uses Unity `AudioMixer` and `Slider` controls
- Stores user volume preferences in `PlayerPrefs`
- Converts slider values to decibels using logarithmic scaling for mixer parameters:
  - `MusicVol`
  - `SFXVol`

## Gameplay Flow

1. Main menu loads and `AudioManager` starts main menu music.
2. Player selects a map in `UIManager.OnMapButtonPressed()`.
3. `MapManager` loads map data and updates obstacles, skybox, and character.
4. UI switches to HUD and `GameManager.StartGame()` begins gameplay.
5. `PlayerMovement` processes input to flip gravity and moves the player forward.
6. `LevelManager` spawns or recycles segments and scales difficulty as distance increases.
7. `DistanceTracker` updates run distance, high score, and unlock states.
8. Colliding with an obstacle triggers `Obstacle.OnTriggerEnter()`, calling `GameManager.GameOver()`.
9. On game over, score is saved and UI transitions to the game over screen.
10. `OnApplicationQuit()` performs a final save before exit.

## Notes and Observations

- Singleton patterns are used heavily in core managers: `GameManager`, `LevelManager`, `MapManager`, `UIManager`, `AudioManager`, `DistanceTracker`, `DataPersistenceManager`
- The project uses Unity's new Input System via `Keyboard.current` and `Pointer.current`
- `PlayerMovement` disables built-in gravity and controls vertical motion manually
- `LevelManager` currently destroys and recreates segments instead of object pooling
- `DataPersistenceManager` does not use `DontDestroyOnLoad`, so scene reloads may reinstantiate the manager
- Obstacle detection includes distance-based triggers and collision-based game over handling
